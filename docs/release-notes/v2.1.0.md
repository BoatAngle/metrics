Metrics now tells you when a new version is out — and the project grew the infrastructure a real open-source app deserves.

## What's new

- **Update check.** Metrics now looks for new releases once a day, so you'll actually hear about fixes like the 2.0.1 energy patch. It's a single anonymous request to GitHub Releases — the only request the app ever makes on its own — and you can turn it off in **Settings → About**. When a new version is out, a quiet banner appears at the top of the dashboard, and **"Check for Updates…"** in any menu-bar item's right-click menu checks on demand.
- **The app now knows its own version** (previous builds reported 1.0.0 internally regardless of release).

## Project improvements

- **MIT license** — the "free and open, fork it" promise is now legally real.
- **Continuous integration** — every push is built, tested, and smoke-tested on GitHub Actions, and tagged releases build their DMG automatically.
- **A test suite** (48 tests) now guards the trickiest pure logic: disk forecasting, alert rules, billing cycles, history rollups, formatters, and version comparison.
- **README refreshed** with the full 2.x feature set, a current screenshot, and an honest, precise Privacy section.

## Installing

1. **Download `Metrics.dmg`** below, open it, and drag **Metrics** to Applications (replace the old copy).
2. If macOS blocks the un-notarized app: **System Settings → Privacy & Security → "Open Anyway"**.

## Compatibility

Universal (Apple Silicon + Intel), macOS 14+.
